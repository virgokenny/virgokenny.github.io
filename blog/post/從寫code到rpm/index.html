目的

最近在公司將我們自己寫的Code要Deploy到RedHat 8或CentOS 8機器上， 所以選擇一些方便的工具來達成目標。

我的目的是可以將這些流程自動化，從Clone Repository、Build Code、跑Unit Test、 包成Rpm Package到建立一個Rpm Repository Server， 這些透過自架的Jenkins和Git Server (Bitbucket)串接。

所以到時候如果機器要安裝或者更新我們的Service都可以透過Rpm Install的方式快速部署。 但由於部署目前專案來說並不是頻率很高的動作，所以部署的工具就先暫不考慮。

Code版本控制

當然就是使用Git來管理，我們是跟Bitbucket買License然後自己架設。 然後我們也有購買Jira，所以在開發流程上就可以將這兩個部分結合讓我們更方便管理我們的專案。

Build Code

因為我們是用C/C++語言，可能有跨平台需求(Windows/Linux)。 所以有考慮CMake或者QMake，只是因為Project並非用QT，而且要用QMake需要安裝QT， 相對CMake來說就有點多餘。

只要善用好CMake可以自動化很多流程，相當方便。

Unit Test

使用C/C++，大概就是用Google Test跟Google Mock了，當然搭配CMake也可以很快速建置Unit Test環境。 但比起其他語言，還是相對麻煩許多。

CI

Jenkins是目前最多人使用的CI Tool，他的Plugin以及資訊也是相當多，免費又可以自建Server， 而Plugin可以跟很多第三方服務做整合(Github、Slack、等等)。 我目前是用一些簡單的Shell Script和Plugin就可以做到很多自動化的工作。

RPM

目前部屬的Server OS主要以RedHat 8和CentOS 8為主，而他們的Package Tool是用RPM， 所以怎麼樣將我們的Code包成RPM是這次比較大的重點。雖然網路上資料很多， 但其中有些細節是真的要碰過才知道。

RPM Repository Server

包好RPM後，就是要讓人可以Install你的RPM。建立好自己的RPM Package Repository Server， 人人就都可以快速安裝或更新你的軟體。

小結

我認為這整套流程是你的Code產品化不可缺少的東西，也許大家的Code語言不同、部屬的OS也不一樣， 但我想整個流程是類似。所以紀錄一下這整套流程並有機會分享一下各個Stage如何做以及可能遇到的問題會是甚麼。
